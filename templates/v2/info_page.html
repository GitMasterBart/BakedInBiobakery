    Transcriptome data shows which genes are activated, and to what extent these genes are active at the time of measurement (Lowe et al., 2017).This can be used to answer specific questions. For example, analyzing the transcriptomic data for multiple samples of interest, makes it possible to see if there are changes or differences in gene expression. When there are changes in an environment, genes can be up- or down regulated. If this is done in a controlled environment, the changes in gene expression can be related to the changes in the environment. In this way, the influence of substances on organisms in an environment can be investigated. Transcriptomics data can also be used to compare change or differences in different tissues or time. (Lowe et al., 2017). The most common way to extract transcriptomic data is by RNA-sequencing. In the last decade the count of publications that involve transcriptomic data acquired with RNA-sequencing (RNA-seq) is grown expansional (Lowe et al., 2017). The biggest contributor to this exponential growth of publication that use transcriptomic data acquired by RNA-seq is because RNA-seq has a dynamic range in order of magnitude 5 over its competitors. Also, the cost of sequencing has decreased significand over the last decades and this change made it more available (Pevsner, 2015). Almost all tools that are available, are designed to run form a command-line interface, either in a Unix environment or within the R/Bioconductor statical environment (Lowe et al., 2017). As a possible side effect, this may cause individuals who have no experience or training in working with a command-line interface see it as a deterrence to start using these techniques. Witch could have as a result that a lot of people do not us these techniques. This is unfortunate mainly because the value of these tools are becoming increasingly apparent (Beghini et al., 2021). The purpose of the BakedinBiobakery is to create a workflow that makes it possible for users that do not have any experience with a command-line interface to do a full transcriptomics data analysis based on functional profiling in metagenomics in the form of a Web-user-Interface (WUI).

    File layout
    The raw input data that is accepted by the BakedinBiobakery is a has a specific construction shown in figure1. A few rules must be complied. Otherwise, the workflow would not work correctly. The first layer of tis construction exist of a folder that has a name representing the analysis. The second layer consist of a group of folders that are named after the samples. In the most outer layer, is filled with fastq files. The important thing to realize is that this application uses forward and backward reads to analyze the samples. This means that each sample folder has 2 inner fastq documents, both these files have a name that compliments the samples folder name, but they have both a unique extension: R1 and R2. Because these data files often contain a few gigabits of data, they are compressed. This makes them smaller and therefore easer to work with.

    Starting process
    After you have singed in, you will be redirected to the upload screen. On this page you can choose witch parameters you want to use in your pipeline.

    note: if you decide to not use the --fastqc-end and --fastqc-start as kneaddata variables it will not be possible to evaluate the reads

    After you have decided witch parameter you want to use, the pipeline will start. There are two options: 1) you wil be redirected to a page where the reads can be evaluated. or 2) The complete pipeline will run and the follow text wil be shown: "Great your file is sent to the system, when the chosen tool is finished with all the hard work you wil get a message."

    note: To make fully use of the application all parameters are recommended (only --verdose is really optional)
